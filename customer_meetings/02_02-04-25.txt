--- Allgemeines zum Projekt ------
> Es soll ein Standalone Tool entwickelt werden (eigenständig lokal lauffähiges Programm)
> Die Lokation der Codebase ist egal, d.h. Github oder z.B. OTH Gitlab möglich
> Das LLM kann lokal auf einem Server (z.B. AWS) deployed werden, MUSS es aber NICHT
	-> die Antwort auf die Aussage, dass LLM anfragen lokal gehostet länger dauern, war "Joa"
	-> das LLM kann auch fremdgehostet und per API angesprochen werden
> Es wird ein "kleines" UI gefordert, d.h. ein mindestens minimales User Interface
> Das geplante AI-Agent System mit Steuerung konfigurierbarer Funktionen im Backend ist erlaubt
> Ein Accountverwaltungssystem ist zum aktuellen Zeitpunkt nicht vorgesehen
> Die Applikation sollte sich bei einer Beschränkung auf eine Sprache bevorzugt auf Englisch anstatt auf Deutsch fokussieren (wegen des theoretisch größeren Nutzerkreises)
> Es wird für die Beantwortung von Anfragen eine Visualisierung bevorzugt; ggf. auch Tabellen-Informationen (die eine eigene Auswertung ermöglichen) aber diese könnten der Visualisierung die Glaubwürdigkeit nehmen
	-> vermutlich ist hier eine (interaktive) Visualisierung mit einsehbaren Daten, z.B. konkreten Auslastungs-Daten, Prognose-Wahrscheinlichkeiten, Trend-Daten etc., eine entsprechende Lösung
> Die Untersuchung der Anfrage-Verarbeitung durch ein LLM (d.h. hauptsächlich, anstatt nur zur Parameterextraktion und Funktionsauswahl) ist vom Fraunhofer durchaus auch gewünscht

>> Es geht bei dem Projekt mehr um den PoC als das konkrete Problem (des mangelnden Büroplatzes), da das konkrete reale Problem noch nicht existiert und nicht alsbald auftreten wird


--- Über die Anfragen ------
> Mögliche Anfragen an das System können von
	"Wie viele Plätze waren gestern besetzt und wie viele frei?"
  bis hin zu
	"Wie würde sich das Belegungsverhalten ändern, wenn anstelle einer Anwesenheits-Policy von einem Tag zwei Tage in der Woche gefordert werden?"
> Wenn bei der Beantwortung einer Anfrage Annahmen getroffen werden, dann sind diese gut zu begründen und darzulegen
> Alternativ ist anzugeben, welche weiteren Informationen zur Beantwortung der Anfrage nötig wären/ sind
> Die Anzahl der möglichen Grundfragen ist "wahrscheinlich nicht wirklich höher als 30"
